«Уютный Дом 2026»

Веб-приложение для студенческого городка

  • Учёт студентов и распределение по комнатам
  • Система заявок в службу поддержки
  • Расписание занятий по группам
  • Админ-панель со статистикой и уведомлениями в реальном времени

Node.js · Express · SQLite · Socket.IO

Цель и задачи

  • Цель: автоматизировать управление общежитием и упростить взаимодействие студентов с администрацией.
  • Обеспечить безопасную аутентификацию (сессии, bcrypt-хеши паролей).
  • Дать студенту личный кабинет с информацией о проживании и расписании.
  • Предоставить администратору панель управления с обновлениями в реальном времени.
  • Включить статистику по студентам и заявкам для анализа.

Функциональность

Студент

  • Вход по логину и паролю
  • Личный кабинет с данными о комнате
  • Расписание на неделю по группе
  • Отправка заявок в поддержку
  • Просмотр ответов и даты визита мастера
  • Тёмная / светлая тема

Администратор

  • Управление студентами (CRUD)
  • Массовое добавление по группам
  • Обработка заявок (ответы, статус, мастер)
  • Статистика по студентам и заявкам
  • Уведомления в реальном времени (WebSocket)
  • Поиск и фильтрация по группам

Архитектура и безопасность

  • Один процесс Node.js: HTTP + Socket.IO на общем сервере.
  • Параметризованные SQL-запросы — защита от инъекций.
  • Ограничение неудачных попыток входа (rate limit).
  • Разграничение ролей: requireRole('admin') для API админки.
  • Хеширование паролей через bcrypt (10 раундов).
  • CSP-заголовки через helmet.

Секрет сессии и порт настраиваются через .env.

Демонстрация на защите

  1. npm install → node server.js (порт 3001).
  2. Вход студента (student101_1 / pass01) → личный кабинет → расписание → отправка заявки.
  3. Во второй вкладке: вход админа (admin / 12345) → панель управления → ответ на заявку.
  4. Показать статистику: студенты по группам, заявки по статусу и месяцам.
  5. Показать появление заявки у студента без перезагрузки (Socket.IO).
  6. Выход через /logout — кабинет недоступен без сессии.

Итоги

  • Реализовано полноценное веб-приложение с клиентской и административной частью.
  • Данные централизованы в SQLite; заявки синхронизируются через WebSocket.
  • Заложены основы безопасности: сессии, bcrypt, rate limit, CSP-заголовки.
  • Адаптивный дизайн с поддержкой тёмной темы.
  • Система статистики для анализа данных общежития.
