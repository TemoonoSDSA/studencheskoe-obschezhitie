1. Теоретические основы разработки программного модуля 

1.1 Анализ предметной области 

Студенческое общежитие представляет собой специализированный объект недвижимости, предназначенный для временного проживания студентов, обучающихся в образовательных учреждениях среднего и высшего профессионального образования. В Российской Федерации функционирует значительное количество студенческих общежитий, которые обеспечивают жильём сотни тысяч обучающихся. Эффективное управление таким объёмом жилого фонда требует внедрения современных информационных технологий и автоматизации административных процессов. 

Анализ предметной области показывает, что основными процессами, требующими автоматизации в студенческом общежитии, являются: 

1. Учёт проживающих студентов - ведение базы данных студентов с информацией о их личных данных, номере комнаты, учебной группе и сроках проживания. 

2. Приём и обработка заявок от студентов - регистрация обращений в службу поддержки по различным вопросам: сантехника, электрика, мебель, пропуск, уборка и другие. 

3. Управление расписанием - ведение расписания занятий по учебным группам, графиков дежурств, субботников и других мероприятий. 

4. Коммуникация между студентами и администрацией - обеспечение своевременного информирования о важных событиях и изменениях в работе общежития. 

5. Статистика и отчётность - формирование отчётов о количестве проживающих, заявках по типам и статусам, посещаемости и других показателях. 

Традиционные методы управления студенческим общежитием основываются на бумажном документообороте и ручной обработке информации. Такой подход характеризуется рядом существенных недостатков: высокая трудоёмкость обработки данных, вероятность ошибок при ручном вводе, сложность поиска необходимой информации, отсутствие оперативного доступа к данным для всех заинтересованных сторон. 

Современные тенденции развития информационных систем в сфере образования предполагают переход к автоматизированным системам управления, которые обеспечивают единое информационное пространство для всех участников образовательного процесса. Внедрение подобных систем в практику работы студенческих общежитий позволяет существенно повысить эффективность административной работы, улучшить качество обслуживания проживающих студентов и обеспечить прозрачность управленческих процессов. 

Анализ существующих решений в области автоматизации управления студенческими общежитиями показывает, что на рынке представлены как специализированные программные продукты, разработанные конкретно для данной предметной области, так и универсальные системы, которые могут быть адаптированы под специфические требования. При выборе программного решения необходимо учитывать масштаб учреждения, численность проживающих, специфику организационной структуры и финансовые возможности. 

Особую актуальность приобретает разработка веб-приложений, которые не требуют установки специализированного программного обеспечения на стороне клиента и обеспечивают доступ к системе через стандартный веб-браузер. Такой подход позволяет организовать удалённый доступ к системе как для студентов, так и для администраторов, что особенно важно в условиях современного образования с развитием дистанционных технологий. 

1.2 Выбор инструментов разработки  

1.2.1 Система управления базами данных 

Выбор системы управления базами данных (СУБД) является одним из ключевых решений при разработке программного модуля для управления студенческим общежитием. СУБД обеспечивает хранение, обработку и извлечение данных, что составляет основу функциональности любого информационного приложения. 

Для разрабатываемого программного модуля была выбрана СУБД SQLite. Данный выбор обусловлен следующими преимуществами: 

1. Простота развёртывания - SQLite является встраиваемой базой данных, не требующей отдельного серверного процесса для своей работы. База данных хранится в одном файле, что существенно упрощает её перенос и резервное копирование. 

2. Отсутствие необходимости в сложной настройке - в отличие от серверных СУБД, таких как PostgreSQL или MySQL, SQLite не требует создания пользователей, настройки прав доступа и конфигурации сетевых параметров. 

3. Низкие требования к ресурсам - SQLite эффективно работает на устройствах с ограниченными вычислительными ресурсами, что важно для образовательных учреждений с ограниченным бюджетом на информатизацию. 

4. Надёжность и устойчивость - SQLite обеспечивает транзакционную целостность данных благодаря механизму журналирования, что гарантирует сохранность информации даже в случае непредвиденного завершения работы приложения. 

5. Достаточная функциональность - для задач программного модуля «Студенческое общежитие» возможности SQLite полностью соответствуют предъявляемым требованиям. Система поддерживает стандартные операции SQL, индексацию, триггеры и представления. 

Структура базы данных разрабатываемого приложения включает следующие основные таблицы: 

- students_groups - содержит информацию о студентах: идентификатор, логин, пароль (в хешированном виде), полное имя, номер учебной группы, номер комнаты, контактные данные; 
- requests - хранит заявки студентов: идентификатор, автор заявки, тип проблемы, текст обращения, дата создания, ответ администратора, дата визита мастера, статус обработки; 
- shower_schedule - график посещения душевых; 
- laundry_bookings - записи на стирку; 
- duty_schedule - график дежурств; 
- subbotniks - информация о субботниках; 
- gym_schedule - расписание работы спортзала. 

Такая структура данных обеспечивает полноценное хранение всей необходимой информации для функционирования системы управления студенческим общежитием. 

1.2.2 Язык программирования и среда разработки программного модуля 

Для реализации программного модуля был выбран язык программирования JavaScript с использованием платформы Node.js и веб-фреймворка Express.js. Данный выбор обусловлен рядом преимуществ данного технологического стека. 

Node.js представляет собой среду выполнения JavaScript, построенную на движке V8 браузера Chrome и позволяющую создавать высокопроизводительные сетевые приложения. Основные преимущества Node.js: 

1. Высокая производительность - асинхронная модель обработки запросов позволяет эффективно обслуживать большое количество одновременных подключений без создания отдельных потоков для каждого запроса. 

2. Единый язык программирования - использование JavaScript как на сервере, так и на клиенте упрощает разработку и поддержку кода, позволяет повторно использовать компоненты и обеспечивает единообразие стиля программирования. 

3. Обширная экосистема - репозиторий npm содержит тысячи готовых библиотек и модулей, которые можно использовать при разработке приложений, что существенно ускоряет процесс создания программных продуктов. 

4. Активное сообщество - Node.js имеет большое и активное сообщество разработчиков, которые создают документацию, учебные материалы и оказывают поддержку начинающим разработчикам. 

Фреймворк Express.js является стандартным решением для создания веб-приложений на Node.js. Он предоставляет минимальный, но гибкий набор функций для построения веб-серверов и API. Основные возможности Express.js, используемые в разрабатываемом приложении: 

1. Маршрутизация запросов - определение путей (роутов) для обработки запросов от клиентов с различными HTTP-методами (GET, POST, PUT, DELETE). 

2. Обработка промежуточного ПО (middleware) - механизм для организации цепочки обработки запросов, позволяющий выполнять различные операции: аутентификация, логирование, валидация данных и другие. 

3. Шаблонизация - поддержка различных движков для генерации HTML-страниц на стороне сервера (в проекте используется отправка статических HTML-файлов). 

4. Статические файлы - встроенная поддержка раздачи статических ресурсов (CSS, JavaScript, изображения). 

Для обеспечения безопасности приложения используются следующие дополнительные модули: 

- bcrypt - библиотека для хеширования паролей с использованием алгоритма Bcrypt, обеспечивающего надёжное хранение аутентификационных данных; 
- express-session - управление пользовательскими сессиями; 
- helmet - набор промежуточного ПО для установки различных HTTP-заголовков, повышающих безопасность приложения; 
- express-rate-limit - ограничение частоты запросов для защиты от атак типа «отказ в обслуживании» и暴力ных подборов паролей. 

Для реализации функциональности реального времени (уведомления о новых заявках) используется библиотека Socket.IO, которая обеспечивает двустороннюю коммуникацию между сервером и клиентами через веб-сокеты. 

Фронтенд приложения реализован на чистом HTML5, CSS3 и JavaScript без использования фреймворков, что обеспечивает: 

1. Простоту понимания и модификации кода; 
2. Быструю загрузку страниц благодаря минимальному объёму передаваемых данных; 
3. Лёгкость развёртывания - статические файлы могут быть размещены на любом веб-сервере или CDN. 

Для визуального оформления интерфейса используется стиль Glassmorphism (эффект матового стекла), который придаёт приложению современный и привлекательный вид. Поддерживается переключение между светлой и тёмной темами оформления. 


2. Практическая часть. Разработка программного модуля «Студенческое общежитие» 

2.1 Описание функциональности программного модуля 

Разработанный программный модуль «Студенческое общежитие» представляет собой веб-приложение, обеспечивающее автоматизацию основных процессов управления студенческим общежитием. Система предоставляет два интерфейса: для студентов и для администраторов. 

Функциональность для студентов включает: 

1. Авторизация в системе - вход по логину и паролю с надёжным хешированием паролей алгоритмом Bcrypt. 

2. Личный кабинет студента - отображение персональной информации: ФИО, учебная группа, номер корпуса и комнаты. 

3. Просмотр расписания занятий - отображение расписания на неделю для учебных групп (101-108). 

4. Отправка заявок в службу поддержки - подача обращений по различным категориям: сантехника, электрика, мебель, пропуск, уборка, другое. 

5. Отслеживание статуса заявок - просмотр своих заявок с ответами администрации и запланированной датой визита мастера. 

6. Управление заявками - возможность скрывать прочитанные заявки. 

7. Восстановление пароля - функция сброса забытого пароля с отправкой нового пароля на email. 

8. Тёмная/светлая тема - переключение между режимами оформления интерфейса. 

Функциональность для администраторов включает: 

1. Управление студентами - добавление, редактирование и удаление записей о студентах. 

2. Массовое добавление студентов - возможность добавления группы студентов (по 10 человек в каждую из 8 учебных групп). 

3. Обработка заявок - просмотр поступивших заявок, отправка ответов, назначение даты визита мастера, изменение статуса обработки (открыта, в работе, выполнена). 

4. Управление видимостью заявок - скрытие и отображение заявок как для студентов, так и в админ-панели. 

5. Поиск и фильтрация - поиск студентов по ФИО и фильтрация по учебным группам. 

6. Статистика - формирование отчётов: количество студентов по группам, заявки по статусам, заявки по типам, динамика по месяцам. 

7. Уведомления в реальном времени - получение уведомлений о новых заявках через Socket.IO без необходимости обновления страницы. 

2.2 Архитектура программного модуля 

Архитектура приложения построена по принципу клиент-серверной модели с использованием REST API. Серверная часть реализована на Node.js с использованием фреймворка Express.js и обрабатывает HTTP-запросы от клиентов. Клиентская часть представляет собой набор статических HTML-страниц со встроенным JavaScript- кодом, который взаимодействует с сервером через AJAX-запросы и WebSocket-соединения. 

Структура проекта включает следующие основные компоненты: 

- server.js - основной файл серверной части, содержащий логику маршрутизации запросов, обработки данных и взаимодействия с базой данных; 
- database/ - каталог с файлом базы данных SQLite; 
- public/ - каталог со статическими файлами фронтенда: 
  - css/ - каскадные таблицы стилей; 
  - html/ - HTML-страницы; 
  - images/ - изображения; 
- logs/ - каталог для хранения логов работы приложения; 
- docs/ - документация проекта. 

Взаимодействие между клиентом и сервером осуществляется по следующим принципам: 

1. HTTP-запросы используются для стандартных операций: получение страниц, отправка данных форм, загрузка и сохранение информации. 

2. WebSocket-соединение (Socket.IO) используется для уведомлений в реальном времени: новые заявки, изменения статусов. 

3. API-эндпоинты предоставляют программный интерфейс для интеграции с внешними системами и автоматизации процессов. 

2.3 Безопасность приложения 

При разработке программного модуля особое внимание уделено обеспечению безопасности. Реализованы следующие меры защиты: 

1. Хеширование паролей - все пароли пользователей хранятся в хешированном виде с использованием алгоритма Bcrypt с 10 раундами соления, что обеспечивает надёжную защиту от несанкционированного доступа к учётным данным. 

2. Защита от SQL-инъекций - все запросы к базе данных используют параметризованные запросы, что исключает возможность внедрения вредоносного SQL-кода через пользовательский ввод. 

3. Ограничение частоты запросов - реализовано ограничение количества попыток входа (5 попыток в течение 15 минут), что защищает от暴力ного подбора паролей. 

4. Управление сессиями - используются защищённые cookies с флагами httpOnly и sameSite, время жизни сессии ограничено 24 часами. 

5. HTTP-заголовки безопасности - через модуль helmet устанавливаются заголовки, защищающие от распространённых атак (XSS, clickjacking и других). 

6. Валидация данных - на стороне сервера выполняется проверка и очистка всех данных, получаемых от пользователей, что предотвращает обработку некорректной информации. 

2.4 Интерфейс пользователя 

Интерфейс программного модуля разработан с учётом современных тенденций веб-дизайна и требований к доступности. Основные характеристики интерфейса: 

1. Адаптивный дизайн - приложение корректно отображается на устройствах с различными размерами экрана: desktop, планшет, смартфон. 

2. Glassmorphism - визуальный стиль с использованием полупрозрачных элементов, размытия фона и плавных градиентов создаёт современный и привлекательный внешний вид. 

3. Тёмная тема - пользователи могут переключаться между светлым и тёмным режимами оформления в зависимости от своих предпочтений. 

4. Интуитивно понятная навигация - меню и элементы управления организованы логически, обеспечивая быстрый доступ к основным функциям системы. 

5. Визуальная обратная связь - система предоставляет пользователям чёткие сигналы об успешном или неуспешном выполнении операций через цветовые индикаторы и уведомления. 

Главная страница приложения содержит информацию о студенческом общежитии, фотографии корпусов и комнат, контактные данные. Личный кабинет студента отображает персональную информацию и обеспечивает доступ к заявкам и расписанию. Админ-панель предоставляет полный набор инструментов для управления системой. 

2.5 Тестирование и внедрение 

Разработанный программный модуль прошёл этап тестирования, в ходе которого были проверены следующие аспекты: 

1. Функциональное тестирование - проверка корректности работы всех функций системы: авторизация, создание и обработка заявок, управление студентами, формирование статистики. 

2. Тестирование безопасности - проверка устойчивости системы к попыткам несанкционированного доступа, SQL-инъекциям и другим атакам. 

3. Нагрузочное тестирование - определение максимальной производительности системы при одновременном обслуживании большого количества пользователей. 

4. Тестирование совместимости - проверка корректной работы приложения в различных браузерах и на устройствах с различными операционными системами. 

Для развёртывания приложения необходимы: 

1. Node.js версии 16 или выше; 
2. npm (менеджер пакетов Node.js); 
3. Операционная система с поддержкой Node.js (Windows, Linux, macOS). 

Процесс установки включает: 

1. Клонирование репозитория проекта; 
2. Установку зависимостей командой npm install; 
3. Настройку переменных окружения в файле .env; 
4. Запуск сервера командой node server.js. 

Приложение будет доступно по адресу http://localhost:3001. Для администратора создана учётная запись с логином «admin» и паролем «12345». В базе данных предустановлено 80 тестовых студентов (по 10 человек в каждой из 8 групп 101-108). 


Заключение 

В ходе выполнения курсовой работы был разработан программный модуль «Студенческое общежитие», обеспечивающий автоматизацию основных процессов управления студенческим общежитием. 

Разработанное приложение позволяет: 

- вести учёт проживающих студентов с возможностью поиска и фильтрации по учебным группам; 
- обрабатывать заявки студентов в службу поддержки с назначением даты визита мастера и отслеживанием статуса; 
- обеспечивать коммуникацию между студентами и администрацией через уведомления в реальном времени; 
- формировать статистические отчёты о работе общежития. 

Технологический стек приложения (Node.js, Express.js, SQLite, Socket.IO) обеспечивает высокую производительность, надёжность и масштабируемость системы при минимальных требованиях к инфраструктуре. 

Использование современных подходов к разработке веб-приложений, включая асинхронную обработку запросов, REST API и WebSocket-соединения, позволяет создать отзывчивый и удобный интерфейс для всех категорий пользователей. 

Меры безопасности, реализованные в приложении (хеширование паролей, защита от SQL-инъекций, ограничение частоты запросов, управление сессиями), обеспечивают надёжную защиту данных и предотвращают несанкционированный доступ к системе. 

Перспективы дальнейшего развития программного модуля включают: 

- расширение функциональности мобильного приложения; 
- интеграция с системой электронного документооборота образовательного учреждения; 
- добавление модуля оплаты коммунальных услуг; 
- внедрение системы бронирования общественных помещений (спортзал, прачечная, душевые). 

Таким образом, поставленные задачи курсовой работы выполнены в полном объёме. Разработанный программный модуль может быть использован в реальных условиях эксплуатации студенческих общежитий образовательных учреждений среднего профессионального образования.
